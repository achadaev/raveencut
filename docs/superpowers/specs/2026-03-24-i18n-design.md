# RaveenCut ‚Äî i18n / Localisation Design

**Date:** 2026-03-24
**Status:** Approved

## Goal

Add Russian and English UI support to RaveenCut with Russian as the default language and an EN | RU toggle button visible on every screen.

## Approach

Use Python stdlib `gettext` with `.po`/`.mo` files. No new runtime dependencies.

## File Structure

```
locale/
  en/LC_MESSAGES/raveencut.po   ‚Üê English source strings
  en/LC_MESSAGES/raveencut.mo   ‚Üê compiled binary (generated)
  ru/LC_MESSAGES/raveencut.po   ‚Üê Russian translations
  ru/LC_MESSAGES/raveencut.mo   ‚Üê compiled binary (generated)
scripts/
  compile_locale.py             ‚Üê runs msgfmt on all .po files
```

## i18n Module

A small section at the top of `app.py` (after imports, before constants) wraps `gettext`:

```python
import gettext as _gettext

_LANG = "ru"          # default
_translation = None

def set_language(lang: str) -> None:
    global _LANG, _translation
    _LANG = lang
    locale_dir = Path(__file__).parent / "locale"
    _translation = _gettext.translation(
        "raveencut", localedir=locale_dir, languages=[lang], fallback=True
    )
    _translation.install()   # installs _ into builtins

def _() -> ...:  # already installed into builtins by set_language()
    ...

# Called once at startup
set_language("ru")
```

`_()` is installed into `builtins` by `translation.install()`, so every module and worker thread can use it without an import.

## Language Switcher UI

- A `QPushButton` labelled `"EN"` or `"RU"` (showing the *inactive* language) placed in the top-right of both `ImportView` and `MainView`.
- Clicking it calls `set_language()` then `retranslate()` on the current view and updates its own label.
- Both views share a `_lang_btn` widget wired to the same slot via `MainWindow`.

## Retranslation

Each view (`ImportView`, `MainView`) implements `retranslate()` which re-calls `setText()`, `setWindowTitle()`, etc. on every widget that contains a user-visible string. Called once on construction (using current default `"ru"`) and again on each language switch.

## Worker Threads

`AnalysisWorker` and `ExportWorker` emit progress strings via `_()` at emit time. `gettext` is stateless per-call after `install()`, so this is thread-safe.

## Build & Tooling

- `scripts/compile_locale.py` ‚Äî runs `msgfmt` on all `.po` files to produce `.mo` files
- `Makefile` target `make locale` as a convenience alias
- PyInstaller spec gains `--add-data "locale:locale"` to bundle `.mo` files
- `.mo` files committed to the repo (no compile step needed for dev or CI)
- `requirements.txt` and `requirements-dev.txt` ‚Äî no changes (stdlib only)

## Testing

- `tests/test_i18n.py` ‚Äî verifies `set_language("ru")` and `set_language("en")` return expected translations for a sample of keys
- Existing GUI tests remain unaffected (they run with whatever default language is active)
